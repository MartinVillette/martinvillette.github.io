Zombie Survival Game

A 2D top-down zombie survival game built with C++ and SDL2. Created as a summer learning project to explore C++ programming, pointer management, and game development fundamentals.

Prototype

Project Overview

This zombie survival game was developed during summer holidays as a hands-on learning experience with C++ programming and game development. The project focuses on fundamental concepts like pointer management, object-oriented design, and real-time graphics rendering using the SDL2 library. Players navigate through a procedurally generated world while fighting off intelligent zombies that use pathfinding algorithms to hunt them down.
C++ SDL2 A* Pathfinding Object-Oriented Design Game Development

Learning Objectives

This project served as a stepping stone toward building more complex AI systems. The ultimate goal was to gain sufficient C++ experience to later develop an AI village simulation where multiple AI agents (villagers) would interact autonomously within a shared environment, each with their own behaviors and decision-making capabilities.
  • C++ Fundamentals
    Master core C++ concepts including classes, inheritance, and memory management.
  • Pointer Management
    Learn proper pointer usage, memory allocation, and avoiding memory leaks in game development.
  • SDL2 Graphics
    Understand 2D graphics rendering, texture management, and sprite animation systems.
  • A* Pathfinding
    Implement intelligent enemy behavior using A* pathfinding algorithms.

Game Features

  • Top-Down Survival Gameplay
    Navigate through a dangerous world using WASD controls while fighting for survival.
  • Intelligent Zombie AI
    Zombies use pathfinding to hunt the player, with view range and attack mechanics.
  • Dynamic Lighting System
    Real-time lighting effects that follow the player and create atmospheric gameplay.
  • Shooting Mechanics
    Mouse-controlled aiming and shooting with bullet physics and collision detection.
  • Sprite Animation
    Character animations for walking, idle states, and combat actions.
  • Procedural Elements
    Dynamic zombie spawning and terrain generation for varied gameplay.

Technical Implementation

The game architecture follows object-oriented principles with a clear inheritance hierarchy. The Human base class is extended by both Player and Zombie classes, promoting code reuse and maintainability. The Window class manages SDL2 rendering and event handling, while the Map class handles world generation and entity management.
Key technical achievements include implementing A* pathfinding for zombie navigation, dynamic lighting effects using SDL2 blend modes, collision detection systems, and efficient memory management with proper pointer usage. The project demonstrates practical application of C++ concepts in a real-time interactive environment.

Future Plans: AI Village

This zombie game serves as the foundation for a more ambitious project: an AI village simulation. The experience gained from building intelligent zombie behavior, pathfinding systems, and entity management will be applied to create autonomous villagers with complex behaviors, social interactions, and emergent gameplay scenarios.
The planned AI village will feature multiple AI agents (villagers) that autonomously navigate the world, perform tasks, interact with each other, and respond to environmental changes. Each villager will have unique personalities, goals, and decision-making processes, creating a living, breathing virtual community.
